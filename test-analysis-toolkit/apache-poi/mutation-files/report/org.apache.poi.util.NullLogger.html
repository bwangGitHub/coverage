328:     /**
329:      * Logs a formated message. The message itself may contain %
330:      * characters as place holders. This routine will attempt to match
331:      * the placeholder by looking at the type of parameter passed to
332:      * obj1.<p>
333:      *
334:      * If the parameter is an array, it traverses the array first and
335:      * matches parameters sequentially against the array items.
336:      * Otherwise the parameters after <code>message</code> are matched
337:      * in order.<p>
338:      *
339:      * If the place holder matches against a number it is printed as a
340:      * whole number. This can be overridden by specifying a precision
341:      * in the form %n.m where n is the padding for the whole part and
342:      * m is the number of decimal places to display. n can be excluded
343:      * if desired. n and m may not be more than 9.<p>
344:      *
345:      * If the last parameter (after flattening) is a Throwable it is
346:      * logged specially.
347:      *
348:      * @param level One of DEBUG, INFO, WARN, ERROR, FATAL
349:      * @param message The message to log.
350:      * @param obj1 The first object to match against.
351:      */
352:
