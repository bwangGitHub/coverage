 41: /**
 42:  * LocalDate is an immutable datetime class representing a date
 43:  * without a time zone.
 44:  * <p>
 45:  * LocalDate implements the {@link ReadablePartial} interface.
 46:  * To do this, the interface methods focus on the key fields -
 47:  * Year, MonthOfYear and DayOfMonth.
 48:  * However, <b>all</b> date fields may in fact be queried.
 49:  * <p>
 50:  * LocalDate differs from DateMidnight in that this class does not
 51:  * have a time zone and does not represent a single instant in time.
 52:  * <p>
 53:  * Calculations on LocalDate are performed using a {@link Chronology}.
 54:  * This chronology will be set internally to be in the UTC time zone
 55:  * for all calculations.
 56:  *
 57:  * <p>Each individual field can be queried in two ways:
 58:  * <ul>
 59:  * <li><code>getMonthOfYear()</code>
 60:  * <li><code>monthOfYear().get()</code>
 61:  * </ul>
 62:  * The second technique also provides access to other useful methods on the
 63:  * field:
 64:  * <ul>
 65:  * <li>numeric value
 66:  * <li>text value
 67:  * <li>short text value
 68:  * <li>maximum/minimum values
 69:  * <li>add/subtract
 70:  * <li>set
 71:  * <li>rounding
 72:  * </ul>
 73:  *
 74:  * <p>
 75:  * LocalDate is thread-safe and immutable, provided that the Chronology is as well.
 76:  * All standard Chronology classes supplied are thread-safe and immutable.
 77:  *
 78:  * @author Stephen Colebourne
 79:  * @since 1.3
 80:  */
