 31: /**
 32:  * DateTime is the standard implementation of an unmodifiable datetime class.
 33:  * <p>
 34:  * <code>DateTime</code> is the most widely used implementation of
 35:  * {@link ReadableInstant}. As with all instants, it represents an exact
 36:  * point on the time-line, but limited to the precision of milliseconds.
 37:  * A <code>DateTime</code> calculates its fields with respect to a
 38:  * {@link DateTimeZone time zone}.
 39:  * <p>
 40:  * Internally, the class holds two pieces of data. Firstly, it holds the
 41:  * datetime as milliseconds from the Java epoch of 1970-01-01T00:00:00Z.
 42:  * Secondly, it holds a {@link Chronology} which determines how the
 43:  * millisecond instant value is converted into the date time fields.
 44:  * The default Chronology is {@link ISOChronology} which is the agreed
 45:  * international standard and compatible with the modern Gregorian calendar.
 46:  * <p>
 47:  * Each individual field can be queried in two ways:
 48:  * <ul>
 49:  * <li><code>getHourOfDay()</code>
 50:  * <li><code>hourOfDay().get()</code>
 51:  * </ul>
 52:  * The second technique also provides access to other useful methods on the
 53:  * field:
 54:  * <ul>
 55:  * <li>numeric value
 56:  * <li>text value
 57:  * <li>short text value
 58:  * <li>maximum/minimum values
 59:  * <li>add/subtract
 60:  * <li>set
 61:  * <li>rounding
 62:  * </ul>
 63:  * <p>
 64:  * DateTime is thread-safe and immutable, provided that the Chronology is as well.
 65:  * All standard Chronology classes supplied are thread-safe and immutable.
 66:  *
 67:  * @author Stephen Colebourne
 68:  * @author Kandarp Shah
 69:  * @author Brian S O'Neill
 70:  * @since 1.0
 71:  * @see MutableDateTime
 72:  */
