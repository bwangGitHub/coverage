 20: /**
 21:  * Wraps a <tt>byte</tt> array and provides simple data input access.
 22:  * Internally, this class maintains a buffer read index, so that for the most part, primitive
 23:  * data can be read in a data-input-stream-like manner.<p/>
 24:  *
 25:  * Note - the calling class should call the {@link #available()} method to detect end-of-buffer
 26:  * and move to the next data block when the current is exhausted.
 27:  * For optimisation reasons, no error handling is performed in this class.  Thus, mistakes in
 28:  * calling code ran may raise ugly exceptions here, like {@link ArrayIndexOutOfBoundsException},
 29:  * etc .<p/>
 30:  *
 31:  * The multi-byte primitive input methods ({@link #readUShortLE()}, {@link #readIntLE()} and
 32:  * {@link #readLongLE()}) have corresponding 'spanning read' methods which (when required) perform
 33:  * a read across the block boundary.  These spanning read methods take the previous
 34:  * {@link DataInputBlock} as a parameter.
 35:  * Reads of larger amounts of data (into <tt>byte</tt> array buffers) must be managed by the caller
 36:  * since these could conceivably involve more than two blocks.
 37:  *
 38:  * @author Josh Micich
 39:  */

 43: 	 * Possibly any size (usually 512K or 64K).  Assumed to be at least 8 bytes for all blocks
 44: 	 * before the end of the stream.  The last block in the stream can be any size except zero.

168: 	 * The {@link #_readIndex} of this (the second) block is updated accordingly.
169: 	 * Note- this method (and other code) assumes that the second {@link DataInputBlock}

176: 		_readIndex = secondReadLen;
177: 	}
